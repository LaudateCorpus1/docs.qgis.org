Installateurs QGIS¶

Comment obtenir QGIS ?

QGIS est disponible librement sur Windows, Linux, MacOS X, BSD et Android

Nous recommandons d’installer les paquets de la dernière version du logiciel.

Pour évaluer la version à venir et permettre aux non-développeurs de contribuer au développement, nous fournissons aussi des paquets testing (notez l”avertissement).

Dans la phase de gel des fonctionnalités précédant la sortie d’une version (voir Calendrier de sortie), ces paquets sont effectivement des pré-versions, que nous recommandons fortement aux utilisateurs de tester.

Windows¶

Pour une installation sous Windows, veuillez s’il vous plaît vous rendre à la Page Principale d’installation.

Aux nouveaux utilisateurs, nous recommandons les installateurs indépendants.

Les utilisateurs plus expérimentés peuvent utiliser le package OSGeo4W qui permet d’installer plusieurs versions de QGIS en parallèle.

Le dépôt OSGeo4W contient de nombreux logiciels issus de projets OSGeo. QGIS et toutes les dépendances sont inclus, avec le Python, GRASS, GDAL, etc. L’installateur peut installer directement depuis Internet ou juste télécharger tous les paquets nécessaires à l’avance. Les fichiers téléchargés sont gardés dans un répertoire local pour des installations futures. Les étapes sont :

  • Download OSGeo4W Installer (32 bit or 64 bit) and start it

  • sélectionner Installateur avancé,

  • naviguer jusque dans la section Desktop,

  • et choisir parmi les paquets suivants :

    qgis-rel-dev [1] Compilation quotidienne de la prochaine sortie ponctuelle
    Répertoire de la version long terme 3.4.x Madeira LTR qgis-ltr Version

Aux utilisateurs qui ne peuvent pas utiliser OSGeo4W pour quelque raison ou juste le préfèrent, il y a aussi un instantané hebdomadaire de qgis-dev depuis l”installateur indépendant d’OSGeo4W (datant du lundi). Dans la phase de gel des fonctionnalités, ça fait aussi office de version candidate.

Notez l”avertissement.

Note

Ne faites pas aveuglément une installation complète d’OSGeo4W. Installez seulement QGIS et peut-être d’autres composantes que vous aimez. Les dépendances seront automatiquement incluses. Une installation complète intègre des composantes qui requièrent l’addition de dépendances tiers-parties, qu’il faudra installer manuellement. Ces composantes rendent l’installation inutilisable sans l’ajout de leurs dépendances.

Linux¶

La plupart des distributions Linux sépare QGIS en différents packages ; vous aurez sans doute besoin de qgis et qgis-python (pour faire tourner les extensions). Les packages comme qgis-grass (ou qgis-plugin-grass), qgis-server peuvent être ignorés au début, et n’être installés qu’en cas de besoin.

Ci-après, vous trouverez des instructions d’installation par distribution. Pour la plupart des distributions, il y a des instructions pour installer la version stable QGIS et des instructions pour installer une version avancée, en cours de développement (mais rappelez-vous de l”avertissement).

Debian/Ubuntu¶

Les dépôts de logiciels par défaut sous Debian ou Ubuntu conservent souvent les anciennes versions de QGIS.

Pour avoir les nouvelles versions, vous devez ajouter des dépôts alternatifs, par ajout d’une des lignes de commande suivantes dans le fichier /etc/apt/sources.list.

Du fait des différentes dépendances disponibles selon les distributions, notre dépôt principal contient de nombreux paquets pour les différentes versions de Debian et Ubuntu

Sous Ubuntu, nous fournissons également des paquets depuis un dépôt séparé basé sur ubuntugis. Ce dépôt propose des versions de paquets SIG plus à jour que ce que contient Ubuntu lui-même. Si vous désirez celles-là, il vous faut donc aussi inclure ubuntugis-unstable ppa dans votre fichier /etc/apt/sources.list (voir ubuntugis documentation).

Note

Les logiciels d’une version ne sont compilés qu’une seule fois juste après la sortie d’une nouvelle version. En tant qu’instables, les versions non encore officielles de debian (testing) et ubuntugis-unstable peuvent subir des changements dans leurs bibliothèques de modules lors du développement de debian, ubuntu ou ubuntugis-unstable qui tôt ou tard casseront les dépendances dont QGIS a besoin. Dans ce cas vous pouvez soit

  • passer à une version stable de Debian et utiliser les paquets sortis pour cela,
  • attendre la prochaine mise à jour mensuelle (prévue pour 2019-09-13 12:00:00 UTC) qui inclura les dépendances actualisées,
  • utiliser un dépôt journalier (disponible pour la version des deux branches et pour le tronc commun) dont les logiciels sont recompilés régulièrement et qui utiliseront automatiquement les dépendances mises à jour ou
  • Montez votre propre ensemble de paquets (voir INSTALLATION).

Ajoutez les lignes pour l’un des dépôts à votre fichier /etc/apt/sources.list:

deb     *repository* *codename* main
deb-src *repository* *codename* main

Exemple pour la dernière version sous Debian unstable:

deb     https://qgis.org/debian unstable main
deb-src https://qgis.org/debian unstable main

Ensuite tapez la commande suivante pour installer QGIS

sudo apt-get update
sudo apt-get install qgis qgis-plugin-grass

In case you would like to install QGIS Server, type:

sudo apt-get update
sudo apt-get install qgis-server

Note

Veuillez supprimer tous les paquets QGIS et GRASS que vous auriez pu avoir installés depuis d’autres sources avant de procéder à la mise à jour.

En cas d’erreurs de serveur de clés, ajoutez la clé publique du dépôt qgis.org à votre trousseau apt en tapant:

wget -O - https://qgis.org/downloads/qgis-2019.gpg.key | gpg --import
gpg --fingerprint 51F523511C7028C3

Ce qui devrait renvoyer:

pub   rsa4096 2019-08-08 [SCEA] [expires: 2020-08-08]
      8D5A 5B20 3548 E500 4487  DD19 51F5 2351 1C70 28C3
uid           [unknown] QGIS Archive Automatic Signing Key (2019) <qgis-developer@lists.osgeo.org>

Après vérification de l’empreinte, vous pouvez ajouter la clé à apt avec:

gpg --export --armor 51F523511C7028C3 | sudo apt-key add -

Autrement, vous pouvez télécharger la clé depuis un serveur de clés et l’ajouter à apt d’une seule traite (sans vérification manuelle de l’empreinte):

sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-key 51F523511C7028C3

Fedora¶

Obtenez les paquets pour Fedora en tapant:

sudo dnf install qgis python3-qgis qgis-grass qgis-server

Les dépôts de logiciels par défaut sous Fedora conservent souvent les anciennes versions de QGIS.

Pour avoir les nouvelles versions, vous devez ajouter des dépôts alternatifs qui diffèrent selon la version que vous souhaitez installer (stable, LTR ou testing).

Version stable de QGIS¶

Activez le dépôt:

sudo dnf copr enable dani/qgis

Ensuite tapez la commande suivante pour installer QGIS

sudo dnf install qgis python3-qgis qgis-grass

Si vous voulez installer QGIS Serveur (notez qu’il n’est pas courant d’installer les applications client et serveur sur la même machine), tapez :

sudo dnf install qgis-server python3-qgis

Ce dépôt fournit également SAGA 2.3.1 compatible avec le module de Traitement. Il peut être installé avec les lignes de commande suivantes :

sudo dnf install saga

openSUSE¶

Version stable de QGIS¶

Dernière version stable pour openSUSE appelée qgis est disponible pour 13.1, 13.2, Leap_42.1, Leap_42.2 et Tumbleweed (32 and 64bit). Ajoutez le dépôt suivant dans votre gestionnaire d’installation, où <VERSION> est par exemple “openSUSE_Tumbleweed”.

https://download.opensuse.org/repositories/Application:/Geo/<VERSION>/

Toutes les sources contiennent le support de GRASS et de Python.

QGIS LTR (Version maintenue à long terme)¶

Version long-terme pour openSUSE appelée qgis-ltr est disponible pour 13.1, 13.2, Leap_42.1, Leap_42.2 et Tumbleweed (32 and 64bit). Ajoutez le dépôt suivant dans votre gestionnaire d’installation, où <VERSION> est par exemple “openSUSE_Tumbleweed”.

https://download.opensuse.org/repositories/Application:/Geo/<VERSION>/

Toutes les sources contiennent le support de GRASS et de Python.

Version en test de QGIS¶

Une version régulièrement mise à jour à partir du tronc commun qgis, appelée qgis-master, est disponible pour 13.1, 13.2, Leap_42.1, Leap_42.2 et Tumbleweed (32 and 64bit). Ajoutez le dépôt suivant dans votre gestionnaire d’installation, où <VERSION> est par exemple “openSUSE_Tumbleweed”.

https://download.opensuse.org/repositories/Application:/Geo/<VERSION>/

Toutes les sources contiennent le support de GRASS et de Python.

ArchLinux¶

Version stable de QGIS¶

Archlinux est disponible sur le dépôt officiel : : https://www.archlinux.org/packages/community/x86_64/qgis/

Installez avec :

pacman -S qgis

QGIS LTR¶

La version Long Terme de QGIS est disponible dans le dépôt des utilisateurs d’Arch (AUR).

Installez avec yaourt ou un autre gestionnaire de paquets compatible AUR :

yaourt -S qgis-ltr

Pour les bogues et autres comportements, lire les commentaires ici : https://aur.archlinux.org/packages/qgis-ltr/

Version en test de QGIS¶

La version en développement de QGIS est disponible dans le dépôt des utilisateurs d’Arch (AUR).

Installez avec yaourt ou un autre gestionnaire de paquets compatible AUR :

yaourt -S qgis-git

Pour les bogues et autres comportements, lire les commentaires ici : https://aur.archlinux.org/packages/qgis-git/

Mac OS X / macOS¶

Les instructions d’installation sont dans le ReadMe sur le disque image. Les téléchargements se font sur la page de téléchargement.

QGIS courant¶

La version actuelle de QGIS utilise python.org Python 3.6 , version 3.6.5 minimum, la compil « macosx10.9 » - les autres distributions ne sont pas prises en charge. Installez les paquets dans l’ordre indiqué. Notez particulièrement que Python doit être installé avant le paquet GDAL Complete et QGIS, sinon les modules GDAL et autres Python ne fonctionneront pas.

Version stable de QGIS¶

Le logiciel stable utilise Python 2.7 - les autres distributions ne sont pas supportées. Les modules Numpy et Matplotlib sont aussi nécessaires dans l’image du disque. Installez les logiciels dans l’ordre énuméré. Si une version majeure plus ancienne est mise à jour (2.16 or plus vieille), effacez QGIS.app de votre répertoire Applications avant d’installer cette version.

D’autres modules Python pour les extensions à utiliser sont disponibles sur kyngchaos.com.

Installateur indépendant¶

Un installateur indépendant est disponible à https://lutraconsulting.github.io/qgis-mac-packager/.

FreeBSD¶

Version stable de QGIS¶

To install QGIS from binary packages type

pkg install qgis

Or to customize compilation options, you can build it from FreeBSD ports

cd /usr/ports/graphics/qgis
make install clean

QGIS LTR¶

To install QGIS from binary packages type

pkg install qgis-ltr

Or to customize compilation options, you can build it from FreeBSD ports

cd /usr/ports/graphics/qgis-ltr
make install clean

Flatpak¶

Il existe une implémentation expérimentale de la dernière version de QGIS sous flatpak.

Pour les notes générales d’installation de Flatpak sous Linux, voir https://flatpak.org/setup/

QGIS sur Flathub: https://flathub.org/apps/details/org.qgis.qgis

Pour installer

flatpak install --from  https://flathub.org/repo/appstream/org.qgis.qgis.flatpakref

Ensuite, pour exécuter

flatpak run org.qgis.qgis

Pour mettre à jour QGIS sous Flatpak

flatpak update

Fichiers Flathub : https://github.com/flathub/org.qgis.qgis

Android¶

Une version expérimentale est disponible sur google play store.

https://play.google.com/store/apps/details?id=org.qgis.qgis

Avertissement

Il n’y a actuellement pas de prise en charge d’Android 5. La meilleure prise en charge est garantie pour Android 4.3 et 4.4.x. Il s’agit d’un simple port de l’application QGIS Desktop. Il est quelque peu optimisé pour les appareils tactiles mais peut toutefois ne pas être adapté à une utilisation quotidienne. Il existe d’autres applications plus adaptées et plus optimisées pour les appareils tactiles.

Avertissement pour QGIS Testing¶

Avertissement

Les packages de QGIS testing sont fournis pour certaines plateformes en plus de la version stable de QGIS. QGIS testing est une version non sortie du logiciel encore en développement. Ils sont seulement fournis à des fins de test afin que des utilisateurs vérifient dès le début si les bogues ont été résolus et si aucun nouveau bogue n’a été introduit. Bien que nous essayions soigneusement d’éviter des ruptures, il peut pas à tout moment ne pas marcher ou faire de mauvaises choses à vos données. Faites attention. Vous avez été prévenus!
