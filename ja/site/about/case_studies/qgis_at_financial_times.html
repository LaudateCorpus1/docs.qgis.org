フィナンシャル・タイムズグラフィックチームでQGISの使用¶

スティーブン・バーナード（インタラクティブデザインエディタ）インタビュー¶

スティーブンは、フィナンシャル・タイムズでインタラクティブデザインエディタです。彼は Twitter で見つけることができます。彼はまた、QGISで一定の地図製作の作業のために、便利な Youtube上HOWTO動画 を公開しています。

QGIS ：スティーブン、あなたがFTで働いているチームの名前は何ですか？FTであなたのチームが何をしているかについて背景を少し提供してもらえますか？

スティーブ：私たちのチームは現時点では移行期間中でして、当社の新しいデータの可視化エディタのアラン・スミスOBE（以前はONSの）が、再編で単一のグラフィックスのチームを作成しているところです。伝統的に、当社にはWebグラフィックチームと印刷グラフィックチームがありました。印刷チームは、すべての新聞のグラフィックスを担当するだけでなく、それらのWebフレンドリーなバージョンを作成することでしょう。私のチームは、すべてのWebコンテンツのみを担当するだけでなく、インタラクティブなグラフィックスを設計する上でインタラクティブな生産/開発者と緊密に協力しました。

新たに形成されたチームでは、異なるメディア間のスキルの多くのクロスオーバーが可能になるでしょう。

アランは、大きなグラフィック語彙を作成するためD3を使用するとか、ビジュアルを作成するのにかかる時間を減らすことなど、私たちのチームのために単一のデジタルワークフローを導入しました。

例：移民の流れ。西バルカン半島ルート

QGIS ：あなたのチームには何人くらい同僚がいますか？

スティーブン：デザイナー、開発者、生産者と統計学者をすべて含めて24人のチームです

QGIS ：チーム内のあなたの具体的な役割は何ですか？

スティーブン ：私たちは学際的な部門ですので、私はこれといった特定の役割を持っていません。私の役割は、インタラクティブグラフィックスのための応答性のデザインを生産するだけでなく、あまり複雑でインタラクティブガイドにフロントエンドの開発を行うなどです（例えば、 インサイドアイシス株式会社：石油のバレルの旅 ）。また、私は毎日のように、標準のニュース関連のグラフや地図を作成します。過去2年間、私はFTで私たちの地図製作製品の開発に注力してきました。QGISは、私たちが地図を作成する方法変更に尽力してきた、と他のツールと連携して、このようなBlenderは、私たちが地図にアプローチする方法と、より創造的に私を助けてきました。

例：Doraのナイジェリアからシチリアへの旅のアニメーション - 毎年人身売買された女性が何千人もいる

QGIS ：ご自身について少し教えてください。あなたの学歴は何ですか？あなたはFTのデータ視覚化チームに参加する前に何をされていましたか？

例：スイスアルプスのゴッタルド鉄道と道路トンネル

スティーブン：私は科学的なイラストレーターとしての従来どおりの訓練を受けています。私はあなたが尋ねる聞くことは何ですか？これは、医学雑誌、自然史資料のどこに写真ができない視覚的に情報を伝えようとしている何のためにイラストを生産含まれます。人間の循環系を示す図を考える、または壊れた骨を固定するための手順を示します。私が大学を出たときは悲しいことにこうした種類の仕事には多くの求人がなく、基本的にはFTの仕事に落ち着きました。数学は、データの視覚化を扱うとき重要だと思いますが、ずっと得意だったので、それは確かにコーディングでも役に立ちました。

QGIS ：どのようにQGISを発見しましたか？

スティーブン ：私は2013年に私たちが 中東の民族マッピングプロジェクト で働いていたとき、QGISに出くわした

私たちは、コロンビア大学の教授によって作成された地図をジオリファレンスするためにそれを使用しました。最初に私はそれが私が前に17年間のAdobe Illustratorで使用したものとは違っていたためか、追加機能の描画ツールを使用して、それは本当にトリッキーな発見しました！私はまた、プロジェクトを保存するために、もう一度、シェープファイルの編集が完了したときに1回、2回、物事を保存しなければならなかった、それはイライラすることがわかりました。しかし、これは、プログラムで経験不足なだけで、最終的にはそれはまさに私たちが必要とすることができました、それはMapboxベース地図に追加しシェープファイルを作成することでした。

QGIS ：あなたの通常のワークフロー内でのQGISの役割は何ですか？

スティーブン：QGISは今、私たちの図化ワークフローの不可欠な部分です。私は、私たちの地図を作成するために、他のGISツールは使用しません。彼らはAdobe Illustratorの内に留まるより快適に感じるので、私たちのチームのいくつかのメンバーはまた、Map Publisherを使用していますが。私は、弊社の地図の80％が今QGISで生産されていると言うでしょう。

例：米国の公共地。

QGIS ：あなたはQGIS以外の他のどのようなツールを使っていて、これらをどのように組み合わせるのですか？

スティーブン：簡単な位置図にすぎない場合、仕事の95％はQGISで行われ、最終的な研磨がAdobe Illustratorで行われます。

起伏地図をやっている場合、プロセスはたくさんより複雑です。私にはNASAの地球エクスプローラからDEMファイルを取得し、サイズにトリミングするQGISにそれらをもたらすでしょう。私はその後、レンダリング、3Dを作成するためにブレンダーにトリミングされたDEMがかかります。私はhillshading上の衛星オーバーレイを使用していた場合、私にはNASAからBlueMarble画像をダウンロードし、同様にブレンダーにこれを取ります。エクスポートされたブレンダーからレンダリング背景レイヤーとして使用するQGISに戻されます。必要であれば、私は事前にPhotoshopでそれを微調整することがあります。

ＩＳＩＳの石油の物語 の中でのような応答性地図について、私は上記のすべてのツールを使用するが、最後のステップは、ニューヨーク・タイムズのアーチー・ツェによって開発されたオープンソースAdobe Illustratorのスクリプトを使用することです。これによりすべてのデバイス上で動作するマイマップの応答性バージョンを作成できます。

有用な別のツールは、世界中の記者は、地図上に位置をマークできますし、私たちはKMZファイルとしてそれらをダウンロードして、まっすぐQGISにインポートすることができGoogleマイマップです。

私は最近、旅にユーザーを取るためにマイマップをアニメーションに入ってきたように、私が使用して、最終的なツールは、アドビシステムズ社After Effectsです。最近ランドラッシュシリーズで使用されるように：

  • ランドラッシュ・エチオピア
  • ランドラッシュ・ミャンマー
  • ランドラッシュ・インドネシア

QGIS ：QGIS以外の他の空間のツールやGISを使用していますか、そうであれば、どれ？

スティーブン：私のためにすべての方法をQGISが、私の同僚のいくつかの上記と同様、 Map Publisher を使用しています。

例：2016年2月のシリア攻撃

QGIS ：あなたがしなければならないタスクについて、QGISの長所と短所を説明することはできますか？

スティーブン：私が最初に私が唯一の特定のタスクに対処するためのフィット専門ツール、としてそれを見たQGISを使用。私は実際にそれの生産用具表示されませんでした。後半に2014年に私の同僚の二人はシェフィールド大学でアラスデア・レイが保有するQGISコースに行ってきました。２人ともがいかにQGISが良いかについて感激していたので、私は、私たちの地図すべてを製作するのに使用できる道具としてそれを評価する役目を自ら引き受けました。

約2週間以内に私はそれが実行可能になると確信していました。主なセールスポイントは、ストリートレベルのマップのズームレベルのスタイリングなど、すべてのためのスタイルガイドを設定する機能でした。

私たちは名前、緯度、経度、マーカーアイコン、フォントウェイトが、フォントサイズ、およびなたはマーカーが見えたりないかどうかの列で、Googleのシートのセットを持って設定するシステムを持っています。

マーカーアイコンは、そのマーカーについてのすべての詳細を記入VLOOKUPテーブルに接続されたドロップダウンです。

だから私がする必要があることすべては、CSVファイルをダウンロードQGISにインポートし、スタイルを適用することだけで、それで95％の準備ができている地図になっています。

完全に正直に言うと私は何も弱点を思いつきませんが、弱点とか機能不足とは思わないものの、見たり実行できるようになればと思う事は数件あります。

例：中国の都市誘致と移住

QGIS ：QGISで最も好きな概念や機能は何ですか？

スティーブン：私が人生の時間を節約するものとして分類するであろう多数の機能があります！しかし、主にクイックOSMとGeoreferencerにより私は多くの時間を節約してきて、これは期限が厳しいとき絶対に不可欠です。これは、報道機関のために働いているとき、通常90％の時間です。

例：中国の都市誘致と移住。詳細ビュー。

QGIS ：QGISでどのような機能や機能性を最も不足を感じますか？実装されていれば、あなたに大いに役立つでしょう不足している機能のリストをお持ちですか？

スティーブン：将来のバージョンで見たい物事のいくつかですが…

ベジェ曲線のサポート、これはGeoreferencerと併せて、特に有用でしょう。スプラインのプラグインがあることは知っていますが、それはベジェ曲線で可能な完全な制御を与えるものではありません。

大きなファイルサイズの画像として保存できること。現在、これはBlenderに取りこむDEMファイルの地理参照された切り抜きを作ることができる唯一の方法です。しかしそれは画面のサイズに制限されます。例えばQGISにロードされた8,000ピクセルDEMファイルを持っていた場合は、HDビデオで使用するために画面よりも大きなサイズで切り抜きを書き出せるようにしたいと思います。

QGIS ：あなたを助けるQGISで特定の機能がない場合、あなたは自分自身でQGISを改良（QGISコアやPythonプラグインでのコーディングで）することを考えますか？または不足している機能を実装するためにQGIS開発者を雇うことを検討しますか？それとも、代わりにそのギャップを埋めるために他のツールを探すのでしょうか？

スティーブン：確かに私は自分でコードを書くには必要なスキルを持っていません。私はJavaScriptはかなり堪能ですがPythonには苦労するでしょう。機能を実装するために開発者を雇えれば素敵ですが、残念ながら私には予算の裁量は何もないので、これを避けるために他のツールを使用しています。

例：米国におけるブロードバンドインターネット加入世帯の割合。

QGIS ：ユーザー体験についてお話しましょう - QGISチームは多くのことを改善できたであろうUIのさまざまな場所（ダイアログ、パネル、など）があることを認識しています。UIで使用すると、UIのオーバーホールのための最大の可能性を見るところについて少し語っていただくことはできますか？

スティーブン ：主なツールは良いと思っていまして、一度慣れたらそれはかなり明白でしょう。しかし、私が少々イライラを感じる一つは、一度プラグインをインストールされていたらそれらを捜し回ることです。これらは、自分のメニュー（MMQGIS）を作成したメニュー（Georeferencer）の1の下で暮らすか、自分のアイコン（threejs）を作成するかどちらかです。インストールしているプラグインがすべて入っているプラ​​グインウィンドウがある便利でしょう。再び、これは約束破りではありませんが、いくつかの異なる場所で捜し回るよりもそれが簡単になるでしょう。

QGIS ：お時間をいただき、またQGISでワークフローを促進し文書化するためにされた仕事について感謝します！

スティーブ・バーナードの肖像。 ファイナンシャル・タイムスのインタラクティブデザインエディタ

スティーブン：喜んで、私はQGISについてほぼ福音宣教です！したがって、 31ビデオチュートリアル と常に言葉を広めることができます。

A.ノイマンとT.サットンによるインタビューの質問。2016年4月。
