撰寫文件¶

QGIS 說明文件是由 ref:Community Team Lead <gui-translation> 管理。請到 QGIS Governance 尋找誰負責引導你。

由 QGIS 文件小組管理的完整文件，可以在 http://www.qgis.org/en/docs/index.html 找到。文件貢獻者清單則可以在這邊 https://github.com/qgis/QGIS-Documentation/graphs/contributors 找到。

說明文件郵件列表¶

如果你計畫要貢獻 QGIS 的文件的話，請看 成為文件撰寫者。有任何相關問題，請聯絡 :ref:`Documentation Team Leader <community-resources>'，或訂閱 `qgis-community-team <http://lists.osgeo.org/mailman/listinfo/qgis-community-team>`_ 郵件清單。

當你訂閱郵件列表後，你可以直接寄信到 qgis-community-team@lists.osgeo.org 尋求協助。

我們強烈建議任何處理文件和翻譯的人加入這個列表，而我們保證這個郵件列表流量相當低。

其他可以訂閱的郵件列表可以在這邊看 郵寄清單。

成為文件撰寫者¶

QGIS 專案一直找尋願意投入更多時間更新 QGIS 文件，以及幫忙翻譯成其他語言的人 - 即便只是協調更新和翻譯都好。

我們嘗試改進我們專案管理的流程，以及分散重擔給不同領域的人，所以你的貢獻會被相當程度的感謝。

由 QGIS 文件小組管理的完整文件，可以在 http://www.qgis.org/en/docs/index.html 找到。所有文件的原始檔都放在 QGIS-Documentation repository。如果你想更新 QGIS 的一些功能，或是撰寫新的文件，你必須：

  • 有 GitHub 帳號 <https://github.com/join>`_
  • 分支程式庫
  • 做你的變動
  • 然後提交 pull request。

貢獻文件和管理的說明用 git 程式庫管理，可以在`文件指南<http://docs.qgis.org/testing/en/docs/documentation_guidelines/index.html>`_找到

小訣竅

直接連接原始文件

要能簡單找到更新過的程式庫，請用 Testing 文件任何頁面下面的 Fix me 連結。這會直接在你的分支 repo 開啟來源文件。你就直接變動、commit 新的分支和提交 pull request 到主要的程式庫。
